## Data files

The `DATA/` folder (downloaded from Zenodo) contains both the **raw simulation outputs** and the **processed/intermediate files** generated by the scripts.

### Raw data files

| File | Description |
|---|---|
| `db_mut_supply.csv` | Every mutation that arose during the adaptation process, with its fitness effect — relative to its parental genotype — classified as beneficial, neutral, deleterious, or lethal. This is the *mutation supply*: the set of mutations the population actually samples from its accessible mutational neighborhood. |
| `db_lineages.csv` | For every surviving genotype (viable organisms present at the end of each experiment), its abundance and its full lineage reconstructed back to the founder genotype. |
| `db_fixed_mutations.csv` | For the fittest surviving lineage of each replicate, the complete evolutionary trajectory of fixed mutations that led to the best-adapted organism by the end of the experiment. |
| `db_mut_load.csv` | Population snapshots taken every 1000 time steps (~100 generations), recording all genotypes coexisting at each sampling point — used to track changes in population composition over time. |
| `db_local_peaks.csv` | For each surviving genotype, the last fixed beneficial mutation and its preceding non-neutral mutation (plus any intermediate neutral mutations) were identified, and the ancestral background of that preceding mutation was reconstructed. All possible single-point mutants of that background were then generated and classified as beneficial/neutral/deleterious/lethal. This characterizes the local fitness-peak structure and is used to determine whether a genotype is *trapped* on a peak (i.e., none of its single-point mutants are beneficial), and whether escaping the peak required epistasis with intermediate neutral mutations. |

### Processed / intermediate data files (generated by the scripts)

| File | Generated by | Description |
|---|---|---|
| `db_all_ancestors.csv` | `1_Extract_most_abundant_organism` | Ancestor reference table (founder sequences not present in `db_mut_supply.csv`). |
| `db_joined_most_abundant_organism.csv` | `1_Extract_most_abundant_organism` | Fitness of the most abundant organism at each sampled update. |
| `db_mut_supply_summary.csv` | `2_Clonal_Interf_Analysis` / `3_mutations_effects_analysis` | Counts/percentages of beneficial, neutral, deleterious and lethal mutations per mutation rate and time bin, summarized from `db_mut_supply.csv`. |
| `db_beneficial_mutations_only.csv` | `2_Clonal_Interf_Analysis` | Subset of `db_mut_supply.csv` containing only beneficial mutations. |
| `db_joined_mut_load_lineages.csv` | `4_mutational_load_lineages_intersection` | Abundance trajectory of the surviving lineages, obtained by joining `db_mut_load.csv` with `db_lineages.csv`. |

## Pipeline overview

The scripts fall into two categories:

- **Data-processing scripts** (`1` and `4`): merge and pre-process the raw simulation output files into the summary tables used for plotting and statistics. These only need to be run once; their output `.csv` files are reused by the analysis scripts.
- **Analysis scripts** (`2`, `3`, `5`, `6`): load the pre-processed tables, reproduce the paper's figures, and run the corresponding statistical tests.

Because of these dependencies, the recommended execution order is:

```
1_Extract_most_abundant_organism              (data processing)
2_Clonal_Interf_Analysis                      (analysis – uses output of script 1)
3_mutations_effects_analysis                  (analysis)
4_mutational_load_lineages_intersection       (data processing)
5_abundance_analysis                          (analysis – uses output of script 4)
6_fitness_peaks_analysis                      (analysis)
```

## Script descriptions

### `1_Extract_most_abundant_organism`
**Purpose:** Builds the ancestor reference table and extracts the fitness of the most abundant organism sampled every 1000 updates.
- Reads `DATA/db_fixed_mutations.csv` to build the ancestor file `DATA/db_all_ancestors.csv`.
- Merges `DATA/db_mut_load.csv` (abundance every 1000 updates) with `DATA/db_mut_supply.csv` (mutation supply) and the ancestor file, to recover the fitness of the dominant lineage at each sampled update.
- **Output:** `DATA/db_joined_most_abundant_organism.csv`

### `2_Clonal_Interf_Analysis`
**Purpose:** Reproduces the figures and statistics for the section *"The role of clonal interference in early adaptation"* (Figure 2A–D).
- Figure 2A–B: fitness trajectories and frequency of beneficial mutational supply over time, plus a paired statistical analysis (Shapiro-Wilk + paired t-test/Wilcoxon) comparing low vs. high mutation rate, and an exponential decay model of beneficial supply.
- Figure 2C: evidence of clonal interference (timing/frequency of competing beneficial mutations).
- Figure 2D: fitness gains of fixed beneficial mutations at the same fixation step.
- **Inputs:** `DATA/db_joined_most_abundant_organism.csv`, `DATA/db_mut_supply_summary.csv`, `DATA/db_beneficial_mutations_only.csv`, `DATA/db_fixed_mutations.csv`.

### `3_mutations_effects_analysis`
**Purpose:** Reproduces the figures and statistics for the section *"Beneficial exhaustion drives a shift toward neutral evolution"* (Figure 3A–E).
- Figure 3A: relative frequency of fixed mutation effects (beneficial/neutral/deleterious) over time.
- Figure 3B–E: combined panel on mutation-effect distribution at the end of the experiment, total fixed mutations, surviving genotypes, and mean pairwise Levenshtein distance between genotypes.
- Requires the external `levenshtein` binary to compute pairwise genotype distances.
- **Inputs:** `DATA/db_fixed_mutations.csv`, `DATA/db_lineages.csv`, `DATA/db_mut_load.csv`, `DATA/db_mut_supply_summary.csv`.

### `4_mutational_load_lineages_intersection`
**Purpose:** Merges abundance data with lineage data to recover the abundance trajectory of the surviving evolutionary lineages.
- Joins `DATA/db_mut_load.csv` with `DATA/db_lineages.csv` on founder/replicate/sequence identifiers.
- **Output:** `DATA/db_joined_mut_load_lineages.csv`

### `5_abundance_analysis`
**Purpose:** Reproduces the figures and statistics for the section *"Clonal monopolization restricts the accumulation of neutral mutations"* (Figure 4A–E, Figure 5A–B, and Supplementary Figures 1–4).
- Defines helper functions to plot abundance time courses of all genotypes and of the dominant lineage, annotating the first appearance of beneficial/neutral/deleterious mutations.
- Figure 4A–B: example abundance trajectories (low vs. high mutation rate).
- Figure 4C: mean number of unique genotypes over time.
- Figure 4D: abundance distribution of neutral vs. beneficial mutation events (with unpaired Wilcoxon test).
- Figure 4E / Figure 5A–B: pairwise genotype (Levenshtein) distance trajectories and lineage-competition dynamics.
- Supplementary Figures 1–4: additional replicate-level abundance/diversity trajectories.
- Requires the external `levenshtein` binary.
- **Inputs:** `DATA/db_mut_load.csv`, `DATA/db_joined_mut_load_lineages.csv`.

### `6_fitness_peaks_analysis`
**Purpose:** Reproduces the analyses for the section *"Strictly neutral epistasis unlocks inaccessible adaptive pathways"*.
- Quantifies the percentage of lineages/experiments trapped at local fitness peaks under low vs. high mutation rate, using strict and relaxed peak definitions.
- Classifies the epistatic effect of a previously neutral mutation on the ancestral (non-neutral) background (lethal/beneficial/neutral/deleterious).
- **Input:** `DATA/db_local_peaks.csv`.

## Phylogenetic tree reconstruction (Figure 5)

The phylogenetic tree shown in **Figure 5** is **not** generated by the scripts above. It is built separately with the scripts packaged in `SCRIPTS/Figure_5_Tree.zip` (compressed because it bundles the `spop` files of all replicates — unzip it before use):

- `from_spop_to_input_python.sh`
- `from_input_python_to_newick.py`
- `from_newick_to_annotation.sh`
- `run-script.sh`
- `how-to-run.txt` — contains the exact usage instructions for the pipeline above.

**Folder structure and workflow (once unzipped):**
- `spop_files/high/` and `spop_files/low/` contain the raw `spop` files for *all* replicates of the high and low mutation-rate treatments, respectively.
- To build the tree for a specific replicate, take the desired `spop` file from `high/` or `low/` and place it inside a new folder named `high_<replicate>_mut_rate` or `low_<replicate>_mut_rate` (e.g. `high_97_mut_rate`, `low_97_mut_rate`), following the convention of the existing example folders.
- When running `run-script.sh`, pass only `high_<replicate>` or `low_<replicate>` (e.g. `high_97`) as the identifier — the `_mut_rate` suffix is used only in the folder name, not as the script argument.
- **Outputs:** a Newick tree file (e.g. `tree_high_97.nwk`, `tree_low_97.nwk`), the corresponding `unique_org_*.csv` table, and the iTOL annotation files (`itol_abundance_*.txt`, `itol_phen_color.txt`, `itol_phen_label.txt`).

**Visualization:** the resulting `.nwk` file was visualized using **iTOL (Interactive Tree Of Life)**, https://itol.embl.de.

## Notes on reproducibility

- Each analysis script ends with a call to `sessionInfo()` to record the exact R version and package versions used to generate the reported results.
- Where raw-data processing is computationally expensive, the original notebooks include commented-out code blocks documenting how the pre-processed summary files were generated, so the full pipeline remains traceable even though only the lighter, pre-processed files are loaded by default.
